# CoAI 电商图片处理 · 执行路线图（分阶段可落地版）

> 日期：2026-06-26
> 来源：基于《2026-06-26-电商图片处理战略调研报告.md》第四/五/六节，逐条拆解为可执行阶段
> 原则：先低风险快赢建立信任 → 再攻护城河（一致性/工作流）→ 后补合规与变现
> 代码基线：前端 `app/src/components/photo/*` + `app/src/hooks/usePhotoTask.ts`；后端 `addition/photo/*` + `adapter/jimengapi/*`

## 阶段总览

| Phase | 主题 | 工期 | 风险 | 依赖 | 价值 |
|---|---|---|---|---|---|
| **P0** | 交互快赢 + 修 bug | 1–2 周 | 低 | 无 | 立竿见影提升体验/信任 |
| **P1** | 商品/模特一致性 + 品牌资产库 | 4–6 周 | 高 | P0 部分 | ⭐ 核心护城河 |
| **P2** | 一键成套素材工作流 + 功能面板重构 | 3–4 周 | 中 | P1 | ⭐ 从工具升级为成套交付 |
| **P3** | 多平台合规 + 批量流水线 + 挑选 | 3–4 周 | 中 | P0/P2 | 出海刚需 + 产能 |
| **P4** | 计费闭环 | 2 周 | 中 | P1/P2 | 留存与变现 |
| **P5** | 图生视频/数字人 + C2PA + URL驱动 + API化 | 4–6 周 | 中 | P2 | 长期布局 |
| **P6** | 画布内局部重绘编辑器 | 3–4 周 | 高 | P0 | 精修刚需（架构级） |

> P0 与 P3 的批量/合规可与 P1 并行启动。P1 是关键路径。P6 架构级独立模块，可在 P1 之后任意插入。

## Phase 0 · 交互快赢 + 修 bug（1–2 周，可立即开工）

**目标**：不改后端模型，补齐"信任三件套"（对比/行内结果/进度文案）并修复 bug。覆盖 **UI 快速见效 1–6**。

### P0.1 修复图库刷新丢失 bug〔UI #4〕
- 现状：`usePhotoTask.ts:46` 初始化只调 `api.listTasks()`，未拉 `listImages`，刷新后图库清空。
- 任务：初始化并发拉 `listTasks()` + `listImages()`；后端缺列表接口则在 `addition/photo/router.go`+`handler.go` 补 `GET /photo/images`（查 `photo_images` 分页）。
- 验收：上传→刷新→缩略图仍在且可继续选中。

### P0.2 任务表行内缩略图 + 悬停浮层〔UI #1〕
- 现状：`TaskTable.tsx:169` 结果默认折叠（`expanded` 才显示）。
- 任务：success 任务行内渲染首图缩略图，悬停放大；保留点击进 Dialog。

### P0.3 before/after 对比滑块〔UI #2〕
- 任务：新增 `BeforeAfterSlider`（拖动分割线对比 source/result），接入 `ResultPreview` Dialog。
- 数据：`PhotoTask` 需带 `source_urls`（核对 `types.go`，缺则补）。

### P0.4 进度上下文化文案〔UI #3〕
- 现状：`TaskTable.tsx:133` 仅百分比进度条。
- 任务：渲染"为 12 张去背，第 5/12 张…"等具体文案（i18n key）。

### P0.5 部分成功 + 失败置顶 + 三档重试〔UI #6〕
- 任务：批量失败子项置顶；提供「重试 / 换参重试 / 跳过」三档。

### P0.6 空状态教学卡片〔UI #5〕
- 任务：空态预置「白底图 / 模特换装 / 批量去水印」一键起跑示例，解决冷启动。

**出口**：6 项合入 main，回归通过，不引入新模型成本。

## Phase 1 · 商品/模特一致性 + 品牌资产库（4–6 周）⭐ 关键路径

**目标**：让"同一商品/模特/品牌"跨场景跨功能保持一致——把 Seedream 4.0 底座变现成护城河。覆盖 **战略 1、2 + UI #11**。

### P1.1 数据模型：商品/模特/品牌身份
- 新增表 `photo_identity`（id, user_id, type[product|model|brandkit], name, ref_image_urls(JSON), seed/feature, meta(JSON), created_at）。
- 后端 `addition/photo/` 增 identity CRUD + router。

### P1.2 一致性生成管线
- `adapter/jimengapi/` 封装"带参考身份"调用（多参考图 + 锁 seed/主体/纹理），沉淀到 `process.go`。
- 验收：同一商品身份 3 个场景出图，主体一致性人工评分通过。

### P1.3 前端：身份库 UI
- 新增「我的商品 / 我的模特」库：建身份（传参考图）→ 选身份再选功能。

### P1.4 虚拟模特库〔战略 2-a〕
- 复用身份机制 type=model，接 `model_image`/试衣，保证同一张脸跨款式。对标 Botika/绘蛙。
- 验收：同一模特换 3 套衣服，脸不变。

### P1.5 品牌资产库 Brand Kit〔战略 2-b，原漏项补回〕
- type=brandkit：用户存 Logo / 主色 / 字体；产出时可叠加锁品牌一致。
- 验收：成套产出中 Logo/色调统一。

### P1.6 参考图拖入定义风格 + 收藏夹资产化〔UI #11〕
- 参考图可拖入定义风格/背景，并收藏为可复用预设（与身份库打通）。
- 验收：收藏的参考风格可一键复用到新任务。

**出口**：商品/模特/品牌身份可存可复用，一致性可量化验收；为 P2 前置。

## Phase 2 · 一键成套素材工作流 + 功能面板重构（3–4 周）⭐

**目标**：上传 1 张图 →（白底+多场景+模特+多平台尺寸+营销图）成套产出。覆盖 **战略 3 + UI #7、#10、#13**。

### P2.1 工作流编排引擎（后端）
- "workflow = 有序 feature 步骤 + 步骤间产物传递"执行器，落在 `processor.go` 之上加编排层；任务表支持父子任务。

### P2.2 预置成套模板
- 内置「服装上架套件」「美妆主图套件」等：固定步骤 + 默认参数 + 绑定身份。

### P2.3 成套结果展示
- 结果按"套"分组（非散列任务行），整套 ZIP 下载（复用 `getDownloadZipUrl`）。

### P2.4 配方保存/复用/分享〔UI #13〕
- 把一套操作（功能链+参数+风格+尺寸）存为配方，带可视预览，可复用/分享。

### P2.5 17 功能场景分组〔UI #7〕
- 现状 `FeaturePanel.tsx:19-37` 平铺无分组。按「基础/场景营销/模特试衣/视频/合规」分组。

### P2.6 参数面板渐进式披露〔UI #10〕
- 高频参数常驻、高级折叠、强度滑块；替代当前全塞一个 Dialog。

## Phase 3 · 多平台合规 + 批量流水线 + 挑选（3–4 周，可与 P1 并行）

**目标**：覆盖 **战略 4、5 + UI #8、#9**。

### P3.1 多平台尺寸预设 + outpaint〔战略 4〕
- 内置淘宝/天猫/京东/拼多多/抖音/Amazon 1:1/TikTok 4:5/Etsy 4:3 预设；用扩图（非裁切）保主体。

### P3.2 批量流水线视图〔战略 5-a〕
- 任务表从"被动轮询日志"升级为流水线：阶段可视、整批进度、失败置顶（复用 P0.5）。
- 可评估：`POLL_INTERVAL` 10s 轮询 → SSE/WebSocket 推送。

### P3.3 自动重命名 + 分平台打包导出〔战略 5-b，原漏项补回〕
- 按 SKU/平台规则自动重命名；按平台分目录打包导出。对标 Photoroom 250 张/批。

### P3.4 配置一次整批套用〔UI #8〕
- 现状 `FeaturePanel.tsx:111` 多功能批量只能逐个弹窗配参。改为一次配置 → 整批套用。

### P3.5 多变体网格挑选 + 收藏择优〔UI #9〕
- 现状最多出 6 张但无挑选工作流。出多张 → 网格挑选 → 收藏择优。

## Phase 4 · 计费闭环（2 周）〔战略 6〕

### P4.1 计量
- 按 feature/出图量埋计费点（`processor.go`），记录消耗。
### P4.2 积分/订阅
- 免费额度获客（每日积分）+ 透明点数/套餐；优先复用站点既有 quota/订阅模块。务必透明不缩水。
### P4.3 前端额度展示
- 出图前预估消耗、余额不足拦截（复用 `friendlyError` insufficient 分支）。

## Phase 5 · 图生视频/数字人 + C2PA + URL驱动 + API化（4–6 周）

### P5.1 图生视频 / 数字人〔战略 7〕
- 强化 `video_gen` + Seedance：商品主图视频、数字人口播入口产品化。
### P5.2 C2PA 合规层〔战略 8〕
- AI 生成内容自动写 C2PA 标注；做成出海"合规即服务"开关（卡 2026-08-02 EU AI Act）。
### P5.3 URL/商品链接驱动〔战略 9，原漏项补回〕
- 贴商品链接 → 自动抓图 → 出整套素材，做极简入口。
### P5.4 API 化 + 操作链编排〔战略 10，原漏项补回〕
- 开放接口 + Agent 友好文档，向 ERP/B 端集成（ToB 留存）。

---

## Phase 6 · 画布内局部重绘编辑器（3–4 周，架构级独立）〔UI #12〕

**目标**：补齐电商精修最大短板。现状 `FeaturePanel.tsx:230` 的 `image_erase` 只靠文字 prompt，无涂抹 mask。

### P6.1 画布与双画笔
- Erase / Restore 双画笔、画布缩放/撤销/重做。
### P6.2 Smart Select 对象级 mask
- 智能选区生成 mask，跟随选区的浮动工具条。
### P6.3 后端局部重绘接入
- mask + prompt 接 `adapter/jimengapi/edit.go`/inpaint 能力，回写结果。
- 验收：框选商品瑕疵 → 局部重绘 → 主体其余不变。
