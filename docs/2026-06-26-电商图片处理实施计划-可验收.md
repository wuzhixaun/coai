# CoAI 电商图片处理 · 实施计划（可执行 / 可验收）

> 日期：2026-06-26
> 配套：高层视图见《2026-06-26-电商图片处理执行路线图.md》；本文件是开发直接照做的任务卡清单
> 任务卡格式：现状 → 步骤 → 改动文件 → 依赖 → 验收清单（全部勾选才算 Done）→ 估时
> 验收原则：每条验收都必须是**可观测、可复现**的（看得到的行为 / 能跑的命令），不接受"代码写了"作为完成标准

---

## 图例与约定

- **估时**单位：人日（d）。前端默认 React 18 + TS + Vite + Radix + Tailwind；后端 Go + Gin。
- **〔UI #n〕/〔战略 n〕**：对应报告条目编号。
- 验收里 `▶` 表示一条可手动复现的操作路径；`✅` 表示期望结果。
- 每个 Phase 设独立分支 `feat/photo-pN-xxx`，子任务可拆 PR。

## 全局完成定义（Definition of Done）

任一任务卡标记 Done 需同时满足：
1. **可演示**：按验收里的 `▶` 路径能现场复现 ✅ 结果。
2. **不回归**：既有 Photo 流程（上传/单功能/批量/任务表）手测通过。
3. **构建通过**：`app` 前端 `npm run build` 无新增报错；后端编译通过。
4. **成本可控**：涉及模型调用的任务，记录单次调用成本与质量评分。
5. **文档同步**：接口/表结构变更更新到对应文档。
